1. System Context

Vizard Arctic — система, которая читает спутниковые данные Sentinel-1 (IceClass + Composite GeoTIFF), заполняет ледовые пропуски (gap filling) с помощью Temporal U-Net и прокладывает оптимальные маршруты судов через Северный морской путь с учётом ледовых классов и confidence.

Temporal U-Net — архитектура

  • Входные каналы: observed(1) + history(1-5) + gap_mask(1) + sin_month(1) + cos_month(1) = 4..9
  • Encoder: 3 уровня ResidualConvBlock (base_ch → 2x → 4x → 8x) с MaxPool2d
  • Bottleneck: MultiScaleBottleneck — 3 параллельных dilation (1,2,3) + fuse + Dropout2d(0.15)
  • Decoder: 3 уровня ConvTranspose2d + skip connections + ResidualConvBlock
  • Головы: class_head (logits, K классов) + conf_head (sigmoid, 0..1)
  • SE-блоки: Squeeze-and-Excitation (reduction=8) на каждом ResidualConvBlock
  • Нормализация: GroupNorm (для 8GB VRAM) или BatchNorm2d
  • Инференс: тайловый (512x512, overlap=64/128), autocrop valid region, model caching

6. Reconstruction Flow

Процесс AI-реконструкции ледовых пропусков: пользователь выбирает сцену → Frontend отправляет POST запрос → Backend создаёт джоб → JobManager запускает Reconstruction Engine → читает IceClass + Composite → определяет gap-маску → получает исторические сцены → U-Net prediction на GPU → генерирует PNG, NPZ, manifest, summary → сохраняет в Storage → обновляет статус джобы → Frontend опрашивает статус и получает манифест.

Режимы реконструкции

РежимМодельOverlapОписание

balancedTemporal U-Nettile_overlap // 2Тайловый инференс с умеренным overlap; fallback на temporal fill если модель недоступна

7. Route Planning Flow

Процесс маршрутизации: пользователь задаёт start/end координаты и класс судна → Frontend отправляет POST /api/v1/routes/solve → Backend загружает route_grid.npz из артефактов слоя → строит cost grid из ice class + palette.cost + land_cost → A* поиск оптимального маршрута + 2 альтернативы (через corridor penalty) → возвращает RouteSolveResponse с primary + alternatives (distance, ETA, risk, confidence).

A* алгоритм

  • Cost grid = ice_class_cost (из palette) + land_cost (из NPZ) + confidence_penalty * (1 - confidence)
  • Vessel multiplier: Arc4=2.2, Arc9=1.0 — более мощные суда платят меньше за лёд
  • 8-connectivity (4 cardinal + 4 diagonal, diagonal weight = sqrt(2))
  • Блокировка клеток с cost >= 8000 (непроходимый лёд/земля)
  • Альтернативные маршруты: corridor_penalty (radius=5+i, factor=1.55+0.15i) на primary path
  • Downsampling: оригинальный raster → 512x512 для скорости A*
